Контекст

Почему это важно

Реальная задача

Нефтяные компании строят геологические модели на основе данных керна — образцов горной породы из скважин. От качества данных зависит точность подсчёта запасов и проектирование добычи.

Проблема

Данные накапливались десятилетиями из бумажных архивов, лабораторий, цифровых систем. При оцифровке неизбежно появляются ошибки. Если их не найти — геологические модели дадут неверные результаты.

Задача команды

Получить «грязный» датасет с 303 записями по 10 скважинам. Найти все ошибки шести типов. Построить инструмент, который автоматически верифицирует любой CSV с данными керна.

Что получите

Опыт работы с реальной промышленной задачей, навык EDA и статистического анализа данных, готовый кейс в портфолио и возможность поступить в магистратуру без вступительных испытаний.

Что искать

6 типов ошибок

Тип 1
Невозможная пористость
Пористость — доля пор в породе. Не может быть отрицательной или больше 1.
~df['porosity'].between(0, 1)
Тип 2
Отрицательная проницаемость
Закон Дарси: поток всегда направлен по градиенту давления. Проницаемость всегда k>0.
df['permeability_mD'] < 0
Тип 3
Смещение столбца
Плотность породы: 2.0–2.9 г/см³. Если density>10 — туда попала проницаемость при оцифровке.
~df['density_gcc'].between(1.5, 3.5)
Тип 4
Нарушение баланса насыщенностей
Нефть и вода не могут занять больше 100% пор: Sw + So ≤ 1 по определению.
water_sat + oil_sat > 1.001
Тип 5
Дублирование записей
Одна строка записана дважды при двойной оцифровке архивных материалов.
df.duplicated(keep=False)
Тип 6 — сложный
Систематическое смещение
Одна лаборатория стабильно даёт другие значения из-за смены оборудования. Обнаруживается только через сравнение групп.
stats.kruskal(*groups)

Проектирование приложения

Как устроен код

Приложение состоит из трёх слоёв. Каждый решает свою задачу и не смешивается с другими — это делает код понятным и легко расширяемым.

UI
Интерфейс — класс MainWindow, методы _build_*
Создаёт окна, вкладки, кнопки, таблицы. Ничего не считает — только отображает.
PyQt6
VZ
Визуализация — класс MplCanvas, методы update_*
Строит графики matplotlib внутри PyQt-окна. После каждого рисования вызывает canvas.draw().
matplotlib
DL
Данные и логика — run_verification(df), load_file()
Принимает DataFrame, возвращает таблицу флагов. Не знает про кнопки, окна и графики.
pandas · scipy

Главное правило: сначала думайте что показать пользователю, потом — как это вычислить. Не наоборот.

Шаг 1
Опишите приложение тремя предложениями
один абзац, без кода
Прежде чем писать строку кода — ответьте: что делает пользователь? Если не можете описать кратко, задача ещё не ясна. В нашем случае: пользователь загружает CSV, видит аномалии, сохраняет отчёт.

Шаг 3
Логика — в отдельную функцию
данные отдельно от интерфейса
run_verification(df) ничего не знает про кнопки — принимает DataFrame и возвращает флаги. Её можно использовать и в Notebook без изменений.
def run_verification(df):
  flags = pd.DataFrame(index=df.index)
  flags['ошибка'] = ~df['porosity'].between(0,1)
  return flags
Шаг 4
Главное окно — QMainWindow
класс как контейнер данных
Данные хранятся как атрибуты класса — все вкладки работают с одними и теми же объектами.
class MainWindow(QMainWindow):
  def __init__(self):
    self.df = None  # датасет
    self.flags = None # флаги
    self.clean = None # чистые
Шаг 5
Интерфейс из блоков — layout
виджеты в раскладках
PyQt укладывает виджеты в layout. Три основных типа:
LayoutКак укладывает
QVBoxLayoutСверху вниз
QHBoxLayoutСлева направо
QGridLayoutСетка (для метрик)
Шаг 6
Matplotlib в окне — MplCanvas
виджет-обёртка для графиков
Графики matplotlib не показываются сами в PyQt. Нужен FigureCanvas — его добавляют в layout как обычный виджет.
canvas = MplCanvas(figsize=(10,5))
layout.addWidget(canvas)
ax = canvas.fig.add_subplot(111)
canvas.draw() # обязательно!
Шаг 7
Подключение кнопок — signals & slots
нажатие вызывает метод
Кнопка испускает сигнал clicked, мы подключаем его к методу. Нажатие — load_file() — обновление всех вкладок.
btn.clicked.connect(self.load_file)
# нажатие → load_file() → update_all()
Шаг 8
_build_* vs update_*
два разных типа методов
Виджеты создаются один раз при запуске (_build_*). Данные обновляются при каждой загрузке файла (update_*). Не путайте — это разная ответственность.
МетодКогда
_build_*Один раз при запуске
update_*При каждой загрузке
Шаг 9
Стили — CSS в PyQt
единый внешний вид
PyQt поддерживает стили в синтаксисе CSS. Все стили собраны в одну строку STYLE в начале файла и применяются к MainWindow.
STYLE = """QPushButton {
  background: #2E75B6;
  border-radius: 6px;
}"""
app.setStyleSheet(STYLE)
Шаг 10
Точка входа — запуск
if __name__ == '__main__'
QApplication управляет событиями окна. MainWindow создаёт интерфейс. app.exec() запускает цикл обработки событий — приложение работает пока окно не закрыто.
if __name__ == '__main__':
  app = QApplication(sys.argv)
  window = MainWindow()
  window.show()
  sys.exit(app.exec())
Советы команде
С чего начать

Начните с минимальной версии — одна кнопка, одна таблица. Потом добавляйте вкладки. Каждую вкладку делает один человек — они не мешают друг другу.

Частая ошибка

Если график не обновляется — вы не вызвали canvas.draw() после рисования. Если данные не видны во вкладке — вы не вызвали update_* после загрузки файла.
